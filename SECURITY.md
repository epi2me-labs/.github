# Security Policy

## Reporting a Vulnerability

The EPI2ME Labs team and community take security issues seriously. We appreciate your efforts to responsibly disclose your findings.

### How to Report

If you discover a security vulnerability in any EPI2ME Labs repository, please report it by contacting us through:

**Contact**: [nanoporetech.com/about/contact](https://nanoporetech.com/about/contact)

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if available)

### What to Expect

- We will acknowledge receipt of your vulnerability report within 5 business days
- We will provide a more detailed response within 10 business days indicating the next steps
- We will keep you informed about the progress towards fixing the vulnerability
- We will notify you when the vulnerability is fixed

### Please Do Not

- Publicly disclose the vulnerability before it has been addressed
- Access or modify data that does not belong to you
- Perform actions that could negatively impact the availability of our services

## Supported Versions

Security updates are provided for the most recent releases of our projects. Please ensure you are using the latest version before reporting an issue.

## Security Best Practices

When using EPI2ME Labs tools and workflows:

- Keep all dependencies up to date
- Follow secure coding practices
- Validate and sanitize all inputs
- Use appropriate access controls
- Review security advisories for dependencies

## Questions

For any questions about security, please contact us through [nanoporetech.com/about/contact](https://nanoporetech.com/about/contact).
